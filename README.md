# PriceComp — 의약품 가격 비교

의약품·건강기능식품 이름을 입력하면 여러 판매처의 가격을 **배송비와 무료배송 조건까지 포함한 최종 결제가** 기준으로 비교·정렬해 주는 웹 앱입니다. GitHub 스타일의 깔끔한 화이트톤 UI로 구성되어 있습니다.

## 주요 기능

- 🔍 의약품 이름 검색 → 판매처별 가격 비교
- 🚚 판매처별 배송 정책 적용 (무료배송 조건 자동 판정)
- 💰 **최종가 = 상품가 + 배송비** 기준으로 자동 정렬, 최저가 하이라이트
- 🧱 검색 결과를 **2열 블럭**으로 표시 (최저가는 상단 전체 폭 강조)
- 👤 **로그인/회원가입** — 계정에 검색 소스 설정을 저장해 다음에도 재사용
- ⚙️ 여러 검색 소스를 추가/관리할 수 있는 설정 패널
- 🧪 API 키 없이도 데모 데이터로 즉시 실행 가능

## 계정 · 보안

- 로그인하면 설정한 검색 소스가 **계정에 저장**되어 다음 방문에도 유지됩니다.
- **계정 비밀번호**는 scrypt로 **해시**만 저장합니다(원문 저장·복호화 불가).
- 소스에 입력한 **아이디/비밀번호/API 키**는 **AES-256-GCM으로 암호화**해 저장하며,
  본인 로그인 시에만 복호화됩니다. 저장 파일이 유출돼도 `APP_SECRET` 없이는 읽을 수 없습니다.
- 운영 환경에서는 반드시 `APP_SECRET` 환경변수를 설정하세요. 미설정 시 개발용 기본 키로 동작합니다.

> `data/users.json`(계정·비밀 정보)은 `.gitignore`에 포함되어 커밋되지 않습니다.

## 실행 방법

```bash
npm install
npm start
# http://localhost:3000
```

기본은 데모 데이터로 동작합니다. 실시간 가격을 쓰려면 네이버 쇼핑 API 키를 설정하세요:

```bash
cp .env.example .env
# .env 파일에 NAVER_CLIENT_ID / NAVER_CLIENT_SECRET 입력
```

## 데이터 소스에 대하여

이 앱은 **공개 검색 API**(네이버 쇼핑 등)를 통해 가격을 조회합니다.

### 쇼핑몰 아이디/비밀번호를 입력받지 않는 이유

처음 아이디어는 "여러 쇼핑몰의 로그인 정보를 입력받아 각 사이트에서 가격을 긁어오자"였습니다. 하지만 다음 이유로 **채택하지 않았습니다.**

1. **보안 위험** — 타인의 계정 비밀번호를 서버가 보관/전송하면 유출 시 피해가 큽니다. 비밀번호는 애초에 저장하지 않는 것이 원칙입니다.
2. **이용약관 위반** — 대부분의 쇼핑몰은 자동 로그인·스크래핑을 약관으로 금지합니다. 계정 정지·법적 문제로 이어질 수 있습니다.
3. **불필요함** — 가격 정보는 로그인 없이도 공개되어 있습니다. 로그인은 가격 비교에 필요하지 않습니다.
4. **깨지기 쉬움** — 로그인 화면·2단계 인증·봇 차단이 바뀌면 매번 동작이 멈춥니다.

그래서 대신 **공식 공개 검색 API 키**를 소스별로 등록하는 방식을 택했습니다. 안전하고, 약관에 맞으며, 안정적입니다.

## 구조

```
PriceComp/
├── server.js                     # Express 서버 + 검색/배송비 계산 API
├── data/
│   ├── mock-products.json        # 데모 상품·가격 데이터
│   └── shipping-policies.json    # 판매처별 배송 정책
└── public/
    ├── index.html                # 화면
    ├── style.css                 # GitHub 화이트톤 스타일
    └── app.js                    # 검색·렌더링·소스 설정 로직
```

## 주의

가격은 참고용입니다. 실제 결제가·배송비는 판매처에서 다시 확인하세요. **일반의약품은 온라인 판매가 제한**되며 약국 방문이 필요할 수 있습니다.
